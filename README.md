# portal-na26wc-guide

## 项目简介

本仓库用于归档与发布一组独立的 HTML 页面。这些页面不依附于特定的域名或网站，而是作为静态资源集合进行管理。

仓库的主要目的是提供一个结构化的存储与访问空间，方便对多个 HTML 文件进行版本控制、检索与分发。

## 目录说明

```
portal-na26wc-guide/
├── pages/          # 存放主要 HTML 页面文件
├── assets/         # 页面引用的静态资源（CSS、JS、图片等）
├── docs/           # 额外的文档或说明文件
└── README.md       # 本文件
```

各目录用途如下：

- `pages/`：核心目录，包含所有需要归档的 HTML 页面，按功能或类别可进一步划分子目录。
- `assets/`：存放页面所需的公共资源，如样式表、脚本文件、字体或图像。
- `docs/`：用于放置与页面内容相关的辅助资料或使用说明。

## 页面归档说明

- 所有 HTML 页面均为独立文件，不依赖外部特定服务器环境。
- 页面之间可能存在引用关系，建议在 `pages/` 内部通过相对路径链接资源。
- 归档时请保持文件命名清晰，避免使用空格或特殊字符。
- 如页面包含外部资源（如第三方库），建议下载后放入 `assets/` 目录，以保证离线可用性。

## 维护说明

- 新增页面时，请将 HTML 文件放入 `pages/` 对应子目录，并在必要时更新本 README。
- 删除或修改页面时，请同步检查相关资源引用是否失效。
- 提交前请确认文件编码为 UTF-8，避免中文显示异常。
- 如有页面使用特殊脚本或样式，建议在文件头部添加简单注释说明用途。
